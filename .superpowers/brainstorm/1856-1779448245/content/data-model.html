Data Model

What gets stored in the database — 5 simple tables

Database Tables
👤 profiles
id, name, email, role (owner or worker), created_at
One row per user. Linked to Supabase login.
💰 fund_transfers
id, worker_id, amount (₹), note, created_at
Each time owner tops up a worker's balance. Balance = sum of these minus sum of expenses.
🏷️ categories
id, name, created_by (worker_id or null), is_global
Global categories have is_global = true (set by owner). Workers can add their own with is_global = false.
🧾 expenses
id, worker_id, category_id, amount (₹), note, image_url, created_at
Each expense a worker records. image_url points to Supabase Storage (optional).
⚙️ settings
low_balance_threshold (₹)
Single row. Owner can change the low-balance alert amount.

How balance is calculated

Worker Balance = Σ fund_transfers − Σ expenses

No separate balance column — it's always computed fresh from the records. This means the history is always accurate.
