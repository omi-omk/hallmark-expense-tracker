3 Approaches to Build This

All use Next.js (Vercel-native). The difference is what handles your database, auth, and images.

A

Next.js + Supabase ⭐ Recommended

One service (Supabase) handles your database, login system, and image storage. Next.js handles the UI and deploys to Vercel. Everything talks to each other out of the box.

Pros

  • One dashboard for everything
  • Generous free tier
  • Excellent beginner docs
  • Built-in auth + storage

Cons

  • Tied to Supabase platform
B

Next.js + Firebase

Google's Firebase handles database, login, and image storage. Very popular, lots of tutorials online.

Pros

  • Very popular, tons of tutorials
  • Good free tier
  • Real-time updates built-in

Cons

  • NoSQL database (harder for financial data)
  • More complex pricing as you grow
  • Harder to write reports/exports
C

Next.js + Neon + Cloudinary + Resend

Best-in-class tools stitched together: Neon for database, Cloudinary for images, Resend for email.

Pros

  • Each service is best-in-class
  • Most flexible

Cons

  • 3–4 separate dashboards to manage
  • More setup complexity
  • Not beginner-friendly
